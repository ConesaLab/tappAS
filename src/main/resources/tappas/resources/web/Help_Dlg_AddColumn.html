Add Annotation Features Column(s) Parameters

To add annotation feature column(s) to the table, you must provide the following information:

  1. Column(s) Section
    Column Title - short column title to display in table
    Use selected annotation source name - check to automatically use the selected feature source as the title for the column; uncheck to set title manually
    Columns - select the annotation feature column(s) to add:
    • Number of unique feature IDs - total number of unique feature IDs contained in the selected feature(s)
    • Feature ID - feature IDs for selected feature(s)
    • Feature name - feature name or description for selected feature(s)

    Be aware that adding annotation feature columns can take up a considerable amount of memory. Do not add long name/description columns, as is the case with GO terms, unless you really need the information.

    Prefix ID/name with feature name - prefix the feature ID and/or name display with the feature name. Only use if the feature IDs can not be distinguished easily among the features included. If checked, for example, it would display "DOMAIN:PF04389" instead of just 'PF04389".

    .
  2. Annotation Feature(s) Section
    Annotation Feature(s) - check to select the annotation feature(s) to include in the column. Note that the tree contains one branch per annotation source and each branch can be expanded to show all the features it contains. You may select all features for a given annotation source or expand and select specific features only. You may ONLY include features from a SINGLE annotation source.
