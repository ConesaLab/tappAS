Feature Diversity Analysis parameters

To run Feature Diversity Analysis (FDA), you must provide the following information:

  1. List of genes section:
    Test list - select the list of genes on which FDA will be performed. Users can select all genes in the project, or provide a list of gene IDs.
    List file - browse directories and select the gene list file. Must be formatted as a single-column .tsv file.

  2. Analysis parameters:
    Name - choose a name for the analysis.
    Analyze by:
    • Category - for the selected annotation feature categories (for example, PFAM), FDA analyses all annotated genes and confers varying status if at least one of the isoforms does not contain it. This may account for one or more of the IDs contained in the gene (one gene may contain several PFAM domains, but variation in one is enough to label PFAM as VARYING for it). Varying status is hence flagged by category, independent of the specific feature ID or IDs that vary in each of the genes.
    • ID - for the selected annotation feature category, each feature ID is evaluated for varying status for all annotated genes. An ID-sensitive count matrix is produced for each feature, indicating the no. of varying and not varying genes. Data drill down is available for this analysis results.

    Check using - criteria used to confer varying status:
    • Fetaure genomic position - varying status is confered when the start and/or end positions of an annotation feature are different across isoforms of a gene. Accounts for partial feature disruption.
    • Feature presence - varying status is confered only if the feature is completely missing for at least one of the isoforms of that gene. Accounts for feature losses.
    Note that, in cases where a feature category can only be evaluated using one criteria (for example, NMD status can only be assessed by presence), these will disappear from the "Select Annotation Feature" menu.

  3. Select Annotation Feature(s) Section
    Annotation Feature(s) - select the annotation feature(s) to include in the FDA. Note that the tree contains one branch per annotation source and each branch can be expanded to show all the features it contains. You may select all features for a given annotation source or expand and select specific features only. You may include any number of features from multiple annotation sources. Note that, for ID-level analysis, only one feature category at a time can be selected.
    Clear All - click to clear all selected features
    Check All - click to check, select, all features
