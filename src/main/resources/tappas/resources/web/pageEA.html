Several enrichment analysis algorithms are implemented in tappAS, and can be run on all available functional annotation databases, performing both traditional GO term, pathway, etc. enrichment analysis as well as enrichments considering any other functions. As in the rest of analysis, users may specify what data type to use - genes, transcripts or CDS - for the analysis. In addition, both the test and background lists to use can be user-defined, however, available application lists are provided (i.e. DE genes, DIU genes...). Users also need to specify annotation categories for testing as well as some analysis package required parameters (see Help button on the bottom of the dialogue for instructions). For all enrichment analysis, several categories can be simultaneously analyzed.



Functional Enrichment Analysis (FEA)

Functional Enrichment Analysis measures the significance of the overrepresentation of a characteristic (GO term annotation, fucntional feature, etc.) in a test list of elements (genes, transcripts or CDS), in comparison with the proportion in which this characteristic is present in a background list. TappAS currently uses the GOSeq R package as the underlying method for Functional Enrichment. You may view the documentation and installation instructions at: https://www.bioconductor.org/packages/release/bioc/html/goseq.html







Functional Enrichment Analysis Parameters


When preparing to run the analysis, all FEA parameters are described in the Help page, which can be accessed via the Help button located on the bottom of the dialog window.

FEA Results

After running the analysis, the FEA results table will be displayed. This table is contained in the project data tab, located in the top tab panel (see image below). Each table row displays the results for a tested feature: Significant - Yes/No, overrepresentation and underrepresentation p-values, and multiple-testing adjusted P-Values. The number of test genes and total genes containing the selected feature are also shown. Data drill down is available for these lists by clicking on the right mouse button.




Functional Enrichment Analysis Results


Results Summary and Enriched Features Chart

After performing a Functional Enrichment, users may click the visualization button on the left hand side panel to display the Results Summary plots, or to produce an Enriched Features Chart summarizing the significance level and the number of genes in the test list for all significant features (bottom panel). Note that this chart can be explored by using the Help button and the visualization options in the left hand side bar.



Enriched Features Cluster Analysis

When performing a Functional Enrichment of GO terms, the option to run Cluster Analysis on the FEA results is provided in the left hand side menu by clicking on the clusters icon. An example of this cluster analysis results can be seen below: the table on the left displays the generated clusters, while the table on the right shows the IDs and descriptions of the GO term nodes for the selected cluster(s). You may select multiple clusters to see their combined nodes. A results summary of the clusters can be displayed on the bottom panel by clicking on the visualization icon on the left hand side menu.

Gene Set Enrichment Analysis (GSEA)

Gene Set Enrichment Analysis tests for the significance of overrepresentation of certain characteristic (for instance, GO terms, or any functional feature in tappAS) by incorporating a ranking metric (for instance, significance) for the gene/transcript/CDS identifiers in the relevant gene list. As previously stated, available application ranked lists are provided, but users may use any custom ranked list file. Several functional categories can be simultaneously run for both implemented methods.





Differential Feature Inclusion Analysis Parameters


When preparing to run the analysis, all GSEA parameters are described in the Help page, which can be accessed via the Help button located on the bottom left of the dialog window. There are two GSEA methods implemented in tappAS:

GOglm

GOglm provides a length bias-corrected approach for GSEA.

You may view the documentation and installation instructions at:
https://github.com/gu-mi/GOglm

MDGSA

MDGSA provides functions to perform GSEA in several genomic dimensions. Note that the MDGSA can be run for a single ranked list (unidimensional), or combining two ranked lists (multidimensional). Importantly, multidimensional MDGSA in tappAS is ONLY enabled for GO term enrichment.

You may view the documentation and installation instructions at:
https://github.com/gu-mi/GOglm



GSEA Results

The GSEA results table is contained in the project data tab, located in the top tab panel (see image below), and will be displayed immediately after running the analysis. Each table row displays the results (i.e. Significant - Yes/No) for a given annotation feature, as well as overrepresentation and underrepresentation test p-values and adjusted P-Values. As in the rest of analysis, more columns may be added using the (+) button on the right hand side of the table.



Gene Set Enrichment Analysis Results

For multidimensional MDGSA, the visualization button will display the "View Multidimensional Enrichment Summary", which will output a list filtered by a user-defined list or a specified significance threshold. This table can be used to produce a multidimensional MDGSA Result Summary visualization by clicking on the visualization button on the left hand side menu. This new graph will appear in the bottom panel. Use its Help button to get more information on this graph.
